Mise en place d'une structure CLIENT-SERVEUR

Objectifs :

    Le but va être de proposer une architecture comprenant :
  • 1 Un parefeu (avec DHCP, DNS, filtrage réseau, routage…) ⬇

    • Voici donc le parefeu fonctionne sur la machine pfSense.

  • 2 clients (1 Windows et 1 Linux) ⬇

    • Les deux clients sont des machines virtuelles (VMs) qui tournent sur VirtualBox.

  • 1 serveur linux (sauvegarde du site web) ⬇

    • Le serveur linux de sauvegarde est une machine virtuelle (VM) qui tourne sur VirtualBox.

  • 1 serveur linux (hébergement site web) ⬇

    • Le serveur linux d’hébergement du site web est une machine virtuelle (VM) qui tourne sur VirtualBox.

L’accès au serveur linux de sauvegarde peut se faire seulement via le réseau interne de la société (soit les deux clients et le serveur web).

Maquette attendue

  • Schéma réseau ⬇
  • Plan d’adressage ⬇
  • Différentes VMs (possibilité de s’interconnecter via switch ou wifi) ⬇
  • Tableau de filtrage ⬇
  • Sauvegarde et restauration fonctionnelle (des fichiers sources du site web) ⬇
  • La sauvegarde doit être automatisée d’une façon ou d’une autre (tous les jours par exemple) ⬇
  • Une documentation pour la restauration ⬇
  • Un site web tournant sur Apache ou Nginx avec le détail du projet (schémas, documentations…) ⬇
